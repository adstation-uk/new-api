# Nginx + Docker + Certbot 自动 HTTPS 运维手册

## 1. 系统概述
本系统采用 Docker 容器化部署，通过 **Nginx** 作为反向代理服务器，配合 **Certbot** 容器实现 Let's Encrypt 证书的**全自动申请与续期**。无需系统级定时任务（crontab），所有自动化逻辑均在容器内部完成。

- **核心组件**：
  - `nginx:alpine`：Web 服务器与反向代理。
  - `certbot/certbot`：证书管理与自动续期工具。
- **部署目录**：`/home/admin/nginx-proxy/`
- **服务对象**：`www.broadscene.ai` (反向代理至本地 AI 服务端口 3346)

## 2. 架构与自动化原理

### 2.1 目录结构
```text
/home/admin/nginx-proxy/
├── docker-compose.yml    # 核心编排文件
├── conf.d/               # Nginx 配置文件 (default.conf)
├── html/                 # ACME 挑战验证目录 (Webroot)
├── logs/                 # Nginx 日志
└── ssl/                  # 【关键】证书存储目录 (挂载点)
    ├── live/             # 当前有效证书软链接
    ├── archive/          # 历史证书备份
    └── renewal/          # 续期配置文件
```

### 2.2 自动续期机制
系统**不依赖**宿主机的 `crontab`。自动化通过 `docker-compose.yml` 中 `certbot` 服务的自定义 `entrypoint` 实现：
1.  容器启动后进入一个无限循环 (`while :; do ...`)。
2.  每次循环先休眠 **12 小时** (`sleep 12h`)。
3.  唤醒后执行 `certbot renew` 命令。
4.  Let's Encrypt 策略：仅在证书过期前 30 天内才会真正续期，否则跳过。
5.  **优势**：即使服务器重启，只要 Docker 服务启动，该容器就会自动恢复“休眠 - 检查”循环，确保证书永不过期。

## 3. 关键配置解析

### 3.1 Docker Compose 配置 (`docker-compose.yml`)
```yaml
services:
  nginx:
    image: nginx:alpine
    container_name: nginx-proxy
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./conf.d:/etc/nginx/conf.d       # 挂载 Nginx 配置
      - ./html:/usr/share/nginx/html     # 挂载 ACME 验证目录
      - ./ssl:/etc/letsencrypt           # 【关键】共享证书卷

  certbot:
    image: certbot/certbot
    container_name: certbot-renew
    restart: always
    volumes:
      - ./ssl:/etc/letsencrypt           # 与 Nginx 共享证书卷
      - ./html:/var/www/certbot          # ACME 验证目录
    # 自定义入口脚本实现自动循环续期
    entrypoint: >
      /bin/sh -c '
      trap exit TERM;
      while :;
      do
        sleep 12h & wait $$!;
        certbot renew --webroot -w /var/www/certbot --quiet;
      done;'
```

### 3.2 Nginx 配置 (`conf.d/default.conf`)
- **HTTP (80端口)**：
  - 开放 `/.well-known/acme-challenge/` 路径供 Certbot 验证域名所有权。
  - 其他所有请求强制 301 重定向至 HTTPS。
- **HTTPS (443端口)**：
  - 加载证书：`/etc/letsencrypt/live/www.broadscene.ai/fullchain.pem`
  - 反向代理：将流量转发至 `http://host.docker.internal:3346` (宿主机运行的 AI 服务)。

## 4. 日常运维操作指南

### 4.1 查看服务状态
检查容器是否正常运行：
```bash
sudo docker ps --format "table {{.Names}}\t{{.Status}}"
```
*预期：`nginx-proxy` 和 `certbot-renew` 状态均为 `Up`。*

### 4.2 手动测试续期 (安全模式)
在不修改任何文件的情况下，模拟续期过程以检查配置有效性：
```bash
sudo docker exec certbot-renew certbot renew --dry-run
```
*成功标志：输出包含 "Congratulations, all renewals succeeded" 或 "Simulated renewal... succeeded"。*

### 4.3 查看实时日志
- **查看 Certbot 续期日志**：
  ```bash
  sudo docker logs -f certbot-renew
  ```
- **查看 Nginx 访问/错误日志**：
  ```bash
  tail -f /home/admin/nginx-proxy/logs/access.log
  tail -f /home/admin/nginx-proxy/logs/error.log
  ```

### 4.4 强制更新证书
如果私钥泄露或需要立即更新证书（慎用，受限于 Let's Encrypt 频率限制）：
```bash
sudo docker exec certbot-renew certbot renew --force-renewal
```
*注意：执行后需重载 Nginx 以应用新证书：*
```bash
sudo docker exec nginx-proxy nginx -s reload
```

### 4.5 服务器重启后的恢复
由于配置了 `restart: always`，通常只需重启 Docker 服务即可：
```bash
# 方法 A: 重启 Docker 守护进程 (推荐)
sudo systemctl restart docker

# 方法 B: 如果容器未自动启动，手动拉起
cd /home/admin/nginx-proxy
sudo docker compose up -d
```

## 5. 故障排查清单

| 现象 | 可能原因 | 解决方案 |
| :--- | :--- | :--- |
| **HTTPS 无法访问** | Nginx 容器未运行或端口冲突 | 检查 `docker ps`，确认 443 端口未被占用。 |
| **证书过期警告** | Certbot 容器停止运行或网络不通 | 检查 `certbot-renew` 日志，确认服务器能访问 `api.letsencrypt.org`。 |
| **502 Bad Gateway** | 后端 AI 服务 (端口 3346) 未启动 | 检查 `ai-api` 容器状态 (`docker ps`)。 |
| **ACME 验证失败** | 80 端口被防火墙拦截或 Nginx 配置错误 | 确保安全组开放 80 端口；检查 `default.conf` 中 location 配置。 |
| **权限拒绝 (Permission denied)** | 宿主机直接操作 `ssl` 目录 | `ssl` 目录属主通常为 root 或容器用户，请使用 `sudo` 或通过容器操作。 |

## 6. 重要提示
- **不要手动修改** `/home/admin/nginx-proxy/ssl/` 下的文件，所有证书操作应通过 `certbot` 容器完成。
- **域名变更**：若需更换域名，需修改 `conf.d/default.conf` 中的 `server_name`，并删除旧证书后重新运行一次 Certbot 申请流程。
- **备份**：定期备份整个 `/home/admin/nginx-proxy/` 目录，特别是 `ssl` 文件夹。
